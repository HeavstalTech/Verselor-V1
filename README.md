### NOTABLE FEATURES 

1. **➭ pair || connect (Multi-Tenancy):** 
    Verselor V1 WhatsApp bot offers something different with its **Multi-Tenancy** system (pair command) which gives it its Multi-Tenant Feature - This command allows a user to connect multiple users to the bot, each with their own instance of the bot, all managed under one server & a single process ("Usage: .pair <number> || .pair 234xxxx")

2. **➭ verselor || botai (AI Advancement):** 
    One of Verselor V1’s most notable command is the *verselor/bot-ai/boatai/bai* command, categorized as Verselor V1’s most useful feature - known for it's expertise on the verselor-v1 bot and it's ability to run most of the bot's commands. It's useful for easy control over the bot. (you can give it a try after deployment - Just Type: "verselor, pull up the bot's menu for me.")

3. **➭ setlanguage || setlang (Multilingual System):** 
    As generally known, many WhatsApp bots out there are only assigned to one language, making it difficult for users who don't understand that language to use the bot - Verselor V1 has addressed this issue by importing a "Multilingual" system that address users to their general language by their set timezone or set language with the "setlang/setlanguage" command - This command is designed to help Non-English users. ("Usage: .setlang fr || .setlang french")


**Verselor V1 solves some problems that many other bots don't.**<br>
**Basically `User → WhatsApp → Baileys → Command Handler → Plugin System → Response`**

# WEB PANEL DEPLOYMENT ![Verified](https://img.shields.io/badge/verified-yes-success) ![Status](https://img.shields.io/badge/status-tested%20%26%20working-brightgreen)
*For users using panels like Pterodactyl, Discord Hosting Sites, or cPanel.*

## Method 1: Using Pair Code
> 1. Download the complete code as a ZIP file.
> 2. Upload the file to your Node.js server/panel.
> 3. Unzip & Enter The "Verselor-V1-main" folder, select all files and move them to the root directory `../`.
> 4. Set the Node version of your hosting environment to version **20x** or later (Recommended).
> 5. Navigate to your server console and start the deployment (Click Start or run `npm install && npm start`).
> 6. Wait approximately 2 minutes for dependencies to install.
> 7. When prompted in the console, **input your WhatsApp number** (Country Code + Number).
> 8. When prompted for **AuthCode**, Enter your authcode wait for it to verify (don't have an AuthCode? Get [Here](https://bots.heavstal.com.ng/dashboard/authcode))
> 9. A Code will appear. Go to WhatsApp → Linked Devices → Link a Device → Link with phone number & Enter the code.

## Method 2: Using QR Code
> ![Verified](https://img.shields.io/badge/verified-yes-success) ![Status](https://img.shields.io/badge/status-tested%20%26%20working-brightgreen)<br>
> 1. [Fork This Repo](https://github.com/HeavstalTech/Verselor-V1/fork)
> 2. Navigate to the root directory and find the `.env` file and click.
> 3. Click the pencil icon (Edit) and change `USE_PAIR_CODE` to `false`
> 
> **Exmaple:**
> ```env
> USE_PAIR_CODE=false
> ```
> 4. Follow steps 1-5 from above (Upload, Unzip, Start).
> 5. A QR Code will appear in the console.
> 6. Go to WhatsApp → Linked Devices → Link a Device and scan the QR code.

<a id="katabump-tip"></a>
> [!TIP]
> ![Verified](https://img.shields.io/badge/verified-yes-success) ![Status](https://img.shields.io/badge/status-tested%20%26%20working-brightgreen)<br>
> If you're deploying on panels like [Katabump](https://dashboard.katabump.com/auth/login#7d6b58), interactive typing might not work. Do this instead:
> 1. [Fork This Repo](https://github.com/HeavstalTech/Verselor-V1/fork)
> 2. Navigate to the root directory and find the `.env` file and click.
> 3. Click the pencil icon (Edit) and add your WhatsApp Number and AuthCode.
> 
> **Example:**
> ```env
> USE_PAIR_CODE=true
> WHATSAPP_NUMBER=2348166546725
> AUTH_CODE=Heavstal_Bots_Your-46-Digit-Code-Here...
> ```
> 4. Commit changes.
> 5. Follow the deployment steps from *METHOD 1*. The bot will automatically read the file and skip the terminal prompts!

# COMMAND LINE INSTALLATION (Termux/VPS) ![Verified](https://img.shields.io/badge/verified-no-yellow) ![Status](https://img.shields.io/badge/status-not%20tested%20%26%20not%20guaranteed-yellow)

## `Requirements`
* [Node.js](https://nodejs.org/en/) (Version 20+)
* [Git](https://git-scm.com/downloads)
* [FFmpeg](https://ffmpeg.org/download.html)
* Any text editor (VS Code, Notepad, etc)

## `For Termux Users`
**Note:** Do not download Termux from the Play Store. Download it from [F-Droid Here](https://f-droid.org/en/packages/com.termux/).

1. **Update System & Install Dependencies:**
```bash
apt update && apt upgrade -y
pkg install git nodejs ffmpeg imagemagick libwebp -y
```

2. **Clone Repository:**
```bash
git clone https://github.com/HeavstalTech/Verselor-V1.git
cd Verselor-V1
```

3. **Install & Start:**
```bash
npm install
npm start
```
*Follow the prompts in the terminal to enter your number and authcode to get the Pairing Code.*

## `For VPS / Ubuntu / Linux`
```bash
sudo apt update && sudo apt upgrade -y
sudo apt install git nodejs ffmpeg imagemagick libwebp -y
git clone https://github.com/HeavstalTech/Verselor-V1.git
cd Verselor-V1
npm install
npm start
```
